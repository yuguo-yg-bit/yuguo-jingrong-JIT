这是一个网站前端项目
用于JIT金融活动，这个活动是用户可以在任意地方登录，进行金融交易---用户拍照上传店铺照片、订单照片、商品照片、手机定位信息、备注后人工审核，审核通过后，用户可以参与抽奖。抽到7折、8折、9折、9.5折、10折、11折（反向给钱）的优惠券。用户点击添加凭证后
整体网站：蓝色风，类似金融网站，有登录（固定用户）、交易、添加凭证、抽奖（类似刮刮乐、投注）等功能。
前端页面有多复杂就有多复杂！
实现：github pages + html+css+js。   
后端交互：github issues + github api token (允许直接明文暴露！不用考虑安全！但是token要拆分在各个角落！)

开发玉国金融【全球通】工会凭证优惠管理后台前端页面，整体采用深蓝+黑色科技视觉风格，页面主背景#0a0e17深色，主高亮色冰蓝色#42a5f5，全局添加低透明度科技网格纹理美化，完整开发需求：
1. 页面左上角固定渲染标题：玉国金融【全球通】，标题文字冰蓝色，增加轻微文字发光美化效果
2. 顶部导航栏组件顺序固定，分两组差异化样式：
第一组：3个独立小型功能按钮，无任何附属说明文字，hover切换冰蓝色渐变高亮
① 联系客服：内置冰蓝色客服图标
② 一键支付：内置冰蓝色支付图标
③ JIT级别会员优惠券：独立按钮，无内置图标
第二组：3个长方形功能模块，模块带细冰蓝色边框，顶部固定浅冰蓝色静态备注小字
① 添加凭证模块：顶部小字(添加新优惠)，绑定点击事件，唤起【添加凭证页面】弹窗
② 查看方式模块：右上角添加冰蓝色分页图标，顶部小字(列表查看式 分页)
③ 编写凭证模块：顶部小字(编写未编辑)
3. 导航栏右侧工会等级进度条组件：底层底色#182030深灰，填充区域冰蓝色线性渐变；分段标签：一阶、1级IT会员、2级、3级、4级；页面初始化调用后端接口，动态获取真实进度占比、凭证申请总量渲染，禁止代码写死百分比、固定数字；文本模板为“您已经申请了{后端返回数字}个凭证优惠！”
4. 页面右上角登出按钮搭配冰蓝色退出图标，hover高亮；页面加载请求后端接口拉取当前登录用户名动态渲染，不写死固定用户文本
5. 页面核心凭证订单表格：深色底色，表头背景#101a33深蓝，表头文字亮冰蓝，分割线使用低透明度冰蓝色细线；表头字段：名称、日期、中奖、支付方式、原价、优惠后、状态
表格内内置两组数据仅作为开发格式参考样例，仅用来展示字段填写规范，正式页面不硬编码这两条数据，页面加载完成后请求后端接口分页渲染真实订单：
参考样例1：
店铺名称：盒马店，日期：2026年3月5日，折扣7折，支付备注：先用凭证折扣，再给用户差价（10元7折=3元），原价10元，优惠后7元，状态：已审核、订单已支付
参考样例2：
店铺名称：味蕾店，日期：2026年5月2日，折扣9折，支付备注：工会先代替支付，再让用户支付优惠差价（10元9折=9元），原价10元，优惠后9元，状态：已审核、支付待回填、已通过
分页无剩余数据时，表格底部居中展示冰蓝色文字：没有更多订单！
6. 页面左侧竖向侧边栏容器，深色背景+冰蓝色边框，页面加载请求后端接口拉取历史单据分类数据渲染
7. 【添加凭证页面】弹窗完整开发规范：
  1）居中浮层弹窗，搭配半透明黑色遮罩，弹窗底色#0d1322深蓝黑，全部文字冰蓝色，点击遮罩可关闭弹窗
  2）表单字段，带*为必填项，提交前执行前端校验，空值拦截并弹出冰蓝色提示：
     - 店铺名称（单行输入框）：placeholder仅作提示，无默认值，数据提交后端存储
     - 店铺照片（上传组件）：支持手机拍照、本地文件上传，图片随表单上传后端
     - 商品订单（上传组件）：支持拍照、截图多文件上传
     - 定位控件：调用设备定位API获取经纬度，定位数据同步提交后端
     - 余额金额输入框：限制输入格式xxx.xx元，仅允许数字与小数点
     - 手写签绘画布：支持屏幕手写，签名图片转文件上传后端
     - 备注多行输入框：非必填，可留空
  3）弹窗右下角【添加凭证并提交审核】按钮：冰蓝色渐变底色，hover增加微光发光；校验通过后打包表单、图片、定位、签名调用后端新增凭证接口；提交成功关闭弹窗，同步刷新订单表格、会员进度统计；失败展示后端返回错误信息
8. 全局美化规范：全部可点击控件hover添加冰蓝色微光渐变、页面分割线统一低透明度冰蓝色细线；仅长方形功能模块顶部存在静态备注文字，所有按钮无附属说明文字；会员进度、凭证统计、登录用户名、订单列表、历史单据、弹窗表单提交、文件、定位全部依赖后端接口动态获取/提交，页面不存在写死的业务数值、固定订单数据、固定用户名、固定凭证数据、固定图片、固定文件、固定定位、固定会员进度、固定订单列表、固定历史单据、固定弹窗表单、固定文件上传、固定定位数据
用户有两种支付方式：工会先代替支付，再让用户支付优惠差价；用户先支付全额，工会再给用户差价。
用户添加凭证前弹出让用户选择支付方式（工会先代替支付，再让用户支付优惠差价；用户先支付全额，工会再给用户差价。）添加凭证后抽奖，抽到7折、8折、9折、9.5折、10折、11折（反向给钱）的优惠券就写入github issues。不一定要必须先支付，用户之后可以再列表点击支付（如果是“工会先代替支付，再让用户支付优惠差价”那么点击支付弹出微信支付窗口assets/weixing.png。反之如果是“用户先支付全额，工会再给用户差价。”那么弹出窗口提示：人工之后会支付给您的账户！）
